Improving Checkout:
Pickup Experience

My Role: Product Design  ·  Research  ·  UI Design  ·  UX Design  ·  UX Writing

Goal: Reduce cart abandonment by presenting the cheapest delivery option as the default in checkout.

Hypothesis: Aligning the default with user preference will shorten decision time and lift conversion.

Why Change the Default?

  • Users reach payment faster when the preferred choice is pre-selected.
  • According to Baymard research, the average cart abandonment rate is 70%, with 39% of shoppers citing extra costs like shipping as the main reason.
  • Surfacing the cheapest option first sets accurate cost expectations and builds trust.
↓ Cart Abandonment Rate
↑ Checkout Conversion

Process

1
Research Reviewed Baymard guidelines on Shipping & Store Pickup. Audited checkout flows of Vila, Adidas, H&M, Mango, and Nike on both desktop and mobile.
2
Quantitative Data Cross-referenced Power BI (weekly view) and Google Analytics (12-month average) to understand which delivery options customers actually preferred.
3
Competitive Analysis Benchmarked checkout experiences across five major fashion retailers to identify best-in-class patterns.
4
Solution Exploration Ideated and prototyped three distinct concepts before landing on the recommended direction.
5
Implementation & Iteration Shipped v1 → collected data → shipped v2 with address validation enhancements and UI refinements.

Competitive Analysis

Audited checkout flows of Vila, Adidas, H&M, Mango, and Nike to understand how leading brands handle delivery default selection, pickup points, and the map overlay pattern.

Key Insights

  • Set the default to what most customers actually pick, based on real usage data.
  • If shipping isn't free, show the cheapest shipping choice first to build trust and reduce shock at payment.
  • Using a map overlay for pickup point selection lowers mental effort and reduces drop-off.

Current vs. Proposed Flow

Mapped the as-is checkout experience against the proposed solution to clearly identify where friction was occurring and communicate the change to stakeholders.

Solution Exploration

Explored three distinct design directions before selecting the final concept based on feasibility, data fit, and user clarity.

Concept A — Tab Switcher

Splits delivery into Delivery and Pickup tabs with minimal UI adjustments.

Concept B — Map Add-On

Keeps the three-part structure but sets standard home delivery as default and adds map view.

Concept C — Tabs + Map (Recommended)

Divides delivery into two clear parts (Delivery and Pickup) with the map view overlay for pickup point selection.

Implementation & Iteration

v1 — Initial Rollout

  • Default set to cheapest delivery option
  • Tabbed selector introduced for Delivery vs. Pickup

v2 — Current Live Version

  • Address validation enhancements for improved accuracy
  • UI adjustments across Delivery and Overlay screens

Results

The updated checkout shows positive improvements across the key metrics we set out to move:

↓ Cart Abandonment
↑ Overall Conversion
