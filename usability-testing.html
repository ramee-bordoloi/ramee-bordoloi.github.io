Usability Testing: Promo Codes
& Loyalty Rewards

My Role: Product Design  ·  Research  ·  UI Design  ·  UX Design  ·  UX Writing   |   Part of the Loyalty Program

Overview

I redesigned the promo code and loyalty experience. Rewards now live inside the bag alongside promo codes, so users can immediately see what they have. Instead of being auto-applied, rewards are selectable — giving shoppers control over which benefits to use and when. The update also introduces a unified promo component, a stackable rewards model, and an informative overlay.

Research Objectives

  • Validate whether users can find, understand, and successfully apply promo codes and loyalty rewards.
  • Identify friction in copy, layout, and interaction design.
  • Assess comprehension of combined rewards (promo + loyalty stacking).
  • Gather actionable feedback to prioritise design, copy, and tech improvements.

Research Questions

  • Can users locate the promo/reward controls without assistance?
  • Do error and success states provide clear enough guidance?
  • How intuitive is stacking (combining) multiple discounts?
  • Does the promo overlay aid or obstruct the checkout flow?
  • Where do first-time users struggle during sign-up and email verification?

Methodology

4 Participants
45min Per Session
5 Test Scenarios
2 Devices Tested

Approach: Moderated, in-person usability sessions on live staging with new promo UI. Both new & returning user flows tested on Desktop and Mobile (Chrome).

Participants: 4 internal colleagues from Customer Ops and Design — Andrea (desktop), Drake (mobile), Samuel (desktop), and Josefina (mobile).

Test Scenarios

1
Promo Code Application — both valid and invalid code entry.
2
Loyalty Rewards Redemption — finding and applying earned rewards.
3
Promo + Reward Combined — stacking multiple discount types in one checkout.
4
Promo Overlay Interaction — navigating the new promo information overlay.
5
First-Time Customer Flow — sign-up, email verification, welcome code application.

Findings

What's Working Well

  • Promo Overlay Placement — Centred, doesn't obstruct the order total view.
  • Primary CTA — New 'Apply Promocode' button reduces reliance on a small × to close the overlay.
  • Manual Reward Selection — Users actively value control over auto-apply behaviour.
  • Stackable Rewards — Once discovered, all four users leveraged combined discounts.
  • Account Creation Flow — Sign-up and password reset were generally smooth across devices.

Key Friction Points

Sign Out Hover Menu

Andrea needed 7 attempts; Samuel needed 3 to trigger sign-out.

→ Replace hover trigger with click, or extend hover delay

Confusing Copy

'Cannot be combined' messaging in WELCOME10 misleads users about actual combinability rules.

→ Update copy; add tooltip clarifying what can and can't stack

Promo Code Input

Hyphenated codes are hard to paste; 'Apply' CTA is frequently overlooked.

→ Auto-strip hyphens on paste; auto-submit on Enter key

Reward Visibility

Josefina searched in My Account first — didn't expect rewards to appear in the bag.

→ Add rewards shortcut in Account navigation with badge count

Mobile Touch Targets

Edge icons are hard to tap on curved-screen mobile devices.

→ Increase padding; shift icons 8–12px inward from edges

Action Items

1
Rework Sign Out Interaction Test static dropdown vs. click-to-open trigger to reduce accidental dismissal.
2
Refine Reward & Promo Copy Especially the combinability language — make it unambiguous.
3
Input Field Enhancements Auto-format promo codes on paste; highlight 'Apply' button state more clearly.
4
Layout Exploration Two-column or tabbed view for Promo vs. Rewards to improve discoverability.
5
Account Shortcut Surface Rewards in My Account with a summary card and notification badge.
6
Mobile Edge Padding Add 8–12px inward shift for corner icons to improve tap reliability.
7
Email Verification Audit Check bounce rates and resend logic — one participant's account never received the verification email.

Impact & Next Steps

  • Planned Validation Round — Re-run all 5 scenarios after implementing quick wins to measure improvement.
  • KPIs to Track — Promo application success rate, average discount value per order, checkout completion time.
